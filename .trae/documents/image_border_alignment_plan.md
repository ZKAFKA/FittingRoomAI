# 图片边框对齐优化实现计划

## 任务分析
用户反映tryon页面上传人像后，图片左右与边框边界对齐，但上下端距离边框边界有一定距离。需要修改图片和边框的适配逻辑，使得图片上下左右都与上传区域的边框保留5rpx左右的距离。

## 实施计划

### [x] 任务1: 分析当前图片布局结构和样式
- **优先级**: P0
- **依赖**: 无
- **描述**:
  - 查看tryon.wxml中上传区域和图片的布局结构
  - 分析tryon.wxss中相关的样式设置，特别是user-image和upload-area的样式
  - 理解当前图片显示的实现逻辑，包括mode属性和尺寸计算
- **成功标准**:
  - 明确了解当前图片布局的实现方式
  - 找出导致图片上下与边框边界不对齐的原因
- **测试要求**:
  - `programmatic` TR-1.1: 检查tryon.wxml中user-image元素的结构和属性
  - `programmatic` TR-1.2: 检查tryon.wxss中user-image和upload-area的样式设置

### [x] 任务2: 修复图片与边框边界对齐问题
- **优先级**: P0
- **依赖**: 任务1
- **描述**:
  - 修改user-image的样式，确保图片上下左右都与上传区域的边框保留5rpx左右的距离
  - 调整图片的显示模式和尺寸计算逻辑
  - 确保修改后图片仍然保持原始比例
- **成功标准**:
  - 图片上下左右都与上传区域的边框保留5rpx左右的距离
  - 图片保持原始比例
  - 图片显示完整，无裁剪
- **测试要求**:
  - `human-judgement` TR-2.1: 上传图片后，视觉检查图片是否与边框边界均匀对齐
  - `programmatic` TR-2.2: 检查修改后的样式代码是否正确实现了5rpx边距

### [x] 任务3: 测试不同尺寸图片的显示效果
- **优先级**: P1
- **依赖**: 任务2
- **描述**:
  - 测试不同尺寸和比例的图片上传后的显示效果
  - 确保所有图片都能正确显示，保持原始比例，并与边框边界均匀对齐
  - 检查在不同屏幕尺寸下的显示效果
- **成功标准**:
  - 不同尺寸和比例的图片都能正确显示
  - 所有图片都与边框边界均匀对齐，保留5rpx左右的距离
  - 在不同屏幕尺寸下显示效果一致
- **测试要求**:
  - `human-judgement` TR-3.1: 上传不同尺寸和比例的图片，检查显示效果
  - `human-judgement` TR-3.2: 在不同屏幕尺寸下查看显示效果

## 预期结果
修复后，tryon页面上传人像后，图片上下左右都与上传区域的边框保留5rpx左右的距离，保持原始比例，显示完整无裁剪，在不同屏幕尺寸下显示效果一致。
